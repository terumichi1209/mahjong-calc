# 麻雀点数計算アプリ 設計書

## 概要

麻雀の手牌を入力すると、役と点数を自動計算するモバイルアプリ。
Expo (React Native) で構築し、iOS/Android 両対応。

## データモデル

### 牌 (Tile)

```ts
type Suit = 'manzu' | 'pinzu' | 'souzu'
type TileValue = 1 | 2 | 3 | 4 | 5 | 6 | 7 | 8 | 9
type Honor = 'east' | 'south' | 'west' | 'north' | 'white' | 'green' | 'red'

type Tile = NumberTile | HonorTile
```

### 面子 (Meld)

```ts
interface Meld {
  type: 'shuntsu' | 'koutsu'
  suit?: Suit      // 数牌の場合
  tiles?: number[] // 数牌の値の配列
  honor?: Honor    // 字牌の場合
}
```

### 手牌分解結果 (ParsedHand)

```ts
interface ParsedHand {
  melds: Meld[]  // 4面子
  pair: Pair     // 1雀頭
}
```

## 手牌パーサーのロジック

1. 14枚の牌をスーツ別のカウント配列と字牌カウントに変換
2. 雀頭候補（2枚ある牌）を総当たりで試す
3. 残り12枚から再帰的に面子を探す（刻子優先 → 順子）
4. 全消化できれば成立

**特殊形（未実装）**
- 七対子: 7種類の対子
- 国士無双: 幺九牌13種 + 1枚

## 点数計算

### 現状

- 符は **30符固定**
- 子ロンのみ対応
- 基本点 = 符 × 2^(翻数+2)
- 点数 = 基本点 × 4（100点未満切り上げ）

### 正式な符計算（未実装）

| 符の種類 | 符数 |
|---------|------|
| 基本符 | 30符（ロン）/ 20符（ツモ） |
| 刻子（中張牌・暗刻） | 4符 |
| 刻子（中張牌・明刻） | 2符 |
| 刻子（幺九牌・暗刻） | 16符 |
| 刻子（幺九牌・明刻） | 8符 |
| 雀頭（役牌） | 2符 |
| 両面/双碰待ち | 0符 |
| 嵌張/辺張/単騎待ち | 2符 |

### 点数区分

| 翻数 | 呼称 |
|------|------|
| 5翻以上 | 満貫（8000点） |
| 8〜10翻 | 跳満（12000点） |
| 11〜12翻 | 倍満（16000点） |
| 13翻以上 | 三倍満（24000点） |
| 役満 | 役満（32000点） |

## 既知のバグ

- `checkChanta` が `checkJunchan` と同一実装になっており、字牌を含む混全帯幺九が正しく判定されない
- 清一色の判定で `tile.suit` を直接参照しているため、字牌が含まれる場合に型エラーになる可能性がある
- 手牌パーサーが刻子優先のため、刻子/順子の組み合わせによっては正しく分解できないケースがある（例: 111234 → 刻子+順子 ではなく誤認する可能性）
